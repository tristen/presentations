1. Rendering maps at Mapbox
Mapbox
2010 ☞ present
- Just focusing on the history of web maps where Mapbox begins - Before Mapbox branched off of Devseed, talk about building maps for -- World Bank -- NPR -- Energy.gov - Using the work of a small number of open source contributors in the space -- OpenLayers
Map rendering has been down a long beaten path
- You work a large amount of data from various sources and formats - Compile it all together and serve it to a client - As an example, the OSM dataset is ~30gigs and you typically see ~0.00172%; - The biggest improvement in data performance is what you choose not to draw - At this stage Google solved slippy maps with z/x/y tiled images - I should note that this was a very important decision and informs even the fast webgl maps we have today. A coordinate system for tiles was huge.

TileMill ☞
Mapbox Studio Classic ☞
Mapbox Studio
- Our tools for map design have followed along its path for awhile.
2011
TileMill
- Familiar with TileMill? It was our first released design studio for creating maps - Supported connecting to custom databases for map data
  • Data ☞ styled raster maps
  • CartoCSS
- CartoCSS is the styling language inspired by CSS. It's an abstraction for writing style attributes in Mapnik - Feels slow today but it was an improvement as it reused and cached tiles. - Optimization came from using .mbtiles. - Talk about .mbtiles quick (small take with you sqlite db) - A blue square means it's blue all the way down
2014
Mapbox Studio Classic
Mapbox Studio Classic was a short-lived project. In many ways it was a hold out until we built Web GL rendering for vector tiles.
  • Data ☞ vector tiles ☞ styled raster maps
  • CartoCSS
- This is where we experiment with Vector Tiles - Data is cached and vector tiles are stored as protocal buffer's - That's like a .pbf file you may have seen (More props to Google) - Small compact binary encoding that can be decoded quickly. - Ridiculously faster than request images. - And becuase of this we can now serve the world out of a box
2015
Mapbox Studio
This is our latest design tool. Online and fast.
  • Data ☞ vector tiles ☞ clientside styling & rendered maps
  • CartoCSS JSON
- Thanks to vector tiles this changes everything and we can use Web GL for rendering - No more CartoCSS. JSON is a readable format and crosses into the realm of both online interactions (JSON is JavaScript) and for manual adjustments (see demos)
